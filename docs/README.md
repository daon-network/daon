# DAON Documentation Site

This is the Jekyll-based documentation site for DAON (Digital Asset Ownership Network), providing comprehensive guides for creators, developers, and platforms implementing creator protection.

## üöÄ Quick Start

### Local Development

1. **Install Dependencies**
   ```bash
   cd docs/
   bundle install
   ```

2. **Run Jekyll**
   ```bash
   bundle exec jekyll serve
   ```

3. **View Site**
   Open http://localhost:4000

### GitHub Pages Deployment

The site automatically deploys to GitHub Pages when changes are pushed to the main branch.

- **Production URL:** https://daon-network.github.io/docs/
- **Custom Domain:** docs.daon.network (when configured)

## üéØ Content Guidelines

### Writing Style
- **Audience-Focused:** Separate technical and non-technical documentation
- **Action-Oriented:** Use clear calls-to-action and next steps
- **Comprehensive:** Cover all skill levels from beginner to expert
- **Consistent:** Use unified terminology and brand voice

### Technical Documentation
- **Code Examples:** Always provide working, copy-paste ready code
- **Multiple Languages:** Show examples in all supported SDKs
- **Error Handling:** Include error scenarios and solutions
- **Testing:** Provide sandbox/testing instructions

### Creator Documentation
- **Non-Technical:** Avoid jargon, explain concepts clearly
- **Visual:** Use diagrams and screenshots where helpful
- **Step-by-Step:** Break complex processes into simple steps
- **Success Stories:** Include real creator experiences

### Adding New Pages

1. **Create Markdown File**
   ```bash
   # For creators
   touch creators/new-guide.md
   
   # For developers  
   touch api/new-sdk.md
   
   # For platforms
   touch platforms/new-integration.md
   ```

2. **Add Jekyll Frontmatter**
   ```yaml
   ---
   layout: default
   title: "Page Title"
   description: "SEO description"
   ---
   ```

3. **Update Navigation**
   Edit `_config.yml` nav_links if needed

### Styling Guidelines

- **Mobile-First:** Responsive design for all devices
- **Accessibility:** WCAG 2.1 AA compliance
- **Performance:** Optimize images and minimize CSS/JS
- **Brand Consistency:** Use DAON brand colors and typography

### Testing

- **Local Testing:** Always test locally before pushing
- **Cross-Browser:** Test in Chrome, Firefox, Safari, Edge
- **Mobile Testing:** Verify mobile responsiveness
- **Link Checking:** Ensure all internal links work

## üöÄ Deployment

### GitHub Pages
- **Automatic:** Pushes to `main` branch auto-deploy
- **Build Time:** ~1-2 minutes
- **Cache:** CDN cache may take 5-10 minutes to update

### Custom Domain Setup
1. Add CNAME file: `echo "docs.daon.network" > CNAME`
2. Configure DNS: `CNAME docs.daon.network daon-network.github.io`
3. Enable HTTPS in GitHub Pages settings
